# Ray Tracer

**Dan Suissa (BX25)**  
**Assignment 1, CSE306: Computer Graphics**  

This project implements a minimal CPU path-tracer in C++ that renders  
(1) a mirror & glass–sphere scene and (2) a BVH-accelerated cat mesh.

---

## Directory contents

- **`.gitignore`** – ignores build artifacts and binaries.  
- **`AABB.h`** – axis-aligned bounding-box class used by the BVH.  
- **`cat`** – compiled executable for the mesh + BVH demo.  
- **`cat.cpp`** – “main” for loading `cat.obj`, building the BVH, and rendering.  
- **`cat.obj`** – Wavefront OBJ mesh of the cat model.  
- **`cat.png`** – sample output image generated by `./cat`.  
- **`CMakeLists.txt`** – CMake build script (if you prefer CMake over manual compile).  
- **`Geometry.h`** – abstract base class and `Hit` record for all shapes.  
- **`MathUtils.h`** – mathematical utilities (`PI`, `sqr()`).  
- **`Ray.h`** – simple ray struct (`origin`, `direction`).  
- **`Sphere.cpp`** – sphere intersection and normal-flipping logic (inner shell).  
- **`Sphere.h`** – declaration of the `Sphere` class.  
- **`spheres`** – compiled executable for the mirror & glass demo.  
- **`spheres.cpp`** – “main” for the three-sphere scene with Fresnel/refraction.  
- **`spheres.png`** – sample output image generated by `./spheres`.  
- **`stb_image.h`** & **`stb_image_write.h`** – 3rd-party single‐header image I/O.  
- **`stb_impl.cpp`** – enables STB’s implementation macros exactly once.  
- **`TriangleMesh.cpp`** & **`TriangleMesh.h`** – triangle-mesh + BVH code.  
- **`Vector.h`** – 3D vector class with `dot`, `cross`, norm, etc.

---

## Usage

Assuming you have the two binaries **`spheres`** and **`cat`** here, simply run:

```bash
./spheres   # renders spheres.png (mirror, solid glass, hollow glass)
./cat       # renders cat.png     (144 k-triangle mesh via BVH)
